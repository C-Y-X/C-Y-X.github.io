故障说明

在升级Win10 专业工作站版 到2004 后发现 VMware 开启时间过长将近两分钟才可以加载完成 卡在初始化页面 如下

同时虚拟机客户端页面显示黑屏 不能正常远程操作如下图


有时间会提醒用户 “连接错误: 无法协商 SSL。” 有时候不会有更多提醒。。弹窗图片待补充

排查过程

因提醒比较模糊 按照提醒进行排查 走很多弯路 如有同样遇到此情况的朋友 可直接查看解决方法 查看是否可解决

  1. 因提示SSL无法协商 第一时间想到是否是因为域名证书 错误。。正好回想到成就传言我所使用Let’s Encrypt OSCP域名访问可能出现问题。更换为 aliyun提供免费的域名在证书 导入新证书后 仍然存在此问题 排除是ssl证书引起报错
  2. 因升级了win10 2004 版本 怀疑为VMware 与win10 2004 本身虚拟化网卡 不兼容 升级VMware至15.5.6 build-16341506 后 仍存在此问题 。后经过降级VMware各版本 仍存在此问题 但发现每次重启后首次打开无问题 启动一些应用娱乐工作 后发现再次出现问题。。初步将下次排查方向定在本机网络 路由上。
  3. 因该错误在网上没有查询到同样状况的网友 感觉可能是自己本身系统设置 或网络设置问题。因本人电脑使用很多相关工具造成虚拟网卡很多 联想到可能是路由错误 分别开始 从 卸载VMware自身虚拟网卡 Zerotier虚拟网卡 等等软件虚拟网卡 仍未解决 但发现 使用网络重置命令后VMware可正常使用。。。定位到为本机网络设置问题。。。关闭所有防火墙 火绒规则后 仍然存在此问题。。。既然知道重置网络netsh winsock reset命令可临时解决此问题 故本着又不是不能用的原则 告一段落（毕竟曾经有几个版本VMware 不能打开虚拟机页面也是sock无法建立 所有放弃继续查找 主观认为可能是VMWare本身软件问题及个人复杂环境冲突引起bug 所有没再研究 后面发现是这次不是VMware的锅。。）

最终本机环境的故障引起原因及解决方法

随着找到了临时解决方法事情本想事情也告一段落

1. 找到引起故障元凶

在玩GTA5时因为网络问题需要上加速器朋友推荐了流星游戏加速器永久免费而且说至少GTA5上使用不错(北京联通线路)加速器确实不错 个人推荐各位不常用加速器的可以使用试试
因为一直没有查找到原因心中始终把这个事情放在心上，，在一次使用加速器时突然想到是否是加速器的加速规则 与VMWare冲突造成 VMware 网络不正常。 经过几次测试后发现 GTA5-EPIC 使用模式5 的时候的规则造成与VMware部分网络规则有冲突 造成VMware连接服务器时有网络错误。。 停止加速后 使用软件自带修复网络功能即可正常使用VMware

群晖moments 缩略图索引生成失败 及无法浏览图片

群晖配件moments 建立缩略图时需要everyone 对/hoems 文件夹有浏览读取修改权限。
因开通多账号 原本将homes 目录权限设置为 administrator、owner 群组具有完全操控权限 user 组拥有读取 写入权限 /homes 下 用户二级目录设置为 admin 及 owner 组 拥有完全权限 其他用户组禁止读取写入
设置好后 发现群晖moments中 无法生成图片索引 无法查看原图 经过排查发现 需要对homes 根目录给everyone 读取及修改权限才能正常使用 其他 drive 配件无此bug影响。 初步判断为 moments索引程序 执行时所用权限为非root权限 当仅仅对 user admininstrator owner 有操作权限时无法正常读取子目录及文件
ps: moments中 共享图库不会因homes权限不对而造成无法读取。初步判定此问题是bug造成。
遭遇环境为 dsm6.20 moments 为1.2.1

Plex Media Server 进程 cpu占用100%

症状体现为 待机状态下 Plex Media Server 进程 占用cpu100% 已知影响版本1.15.6.1079
在官方论坛发现很多用户在1.15.6.1079版本之前也遇到相关问题 已有有网友分析出原因及结局方法

造成bug原因：

在Plex Media Server 与Plex官方服务器通讯时 会自动寻找本地路由upnp服务器 若路由器为openwrt(lede) 梅林等固件 部分miniupnp服务会与 进程通讯出bug 形成死循环 从而形成cpu占用100% 即使使用手动设置远程端口也会有unpu扫描

本人尝试解决方法

  1. 停止路由upnp服务 重启 Plex服务 不过此方法影响其他局域网内upnp使用 最终未使用

  2. 如果为docker方式安装

    1. 可在docker客户机中进行 防火墙设置 拒收所有 1900端口udp 封包 需要进入docker镜像修改 安装iptabales 防火墙 对镜像修改较大 且不保证更新后该bug是否会修复 PS：根据论坛反馈 最近两版本出现的bug 大概一个月前已经有人反馈过 并未更新 所以没使用该方法
